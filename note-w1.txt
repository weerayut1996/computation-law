[OVERVIEW]
Workshop 1 มุ่งเน้นการสร้าง "Modular Data Pipeline" เพื่อเปลี่ยนข้อมูลกฎหมายดิบ 
(Raw Thai IP Law Text) ให้กลายเป็น Structured Data ที่พร้อมสำหรับงาน Deep Learning 
ในลำดับถัดไป (W2-W4)

[KEY COMPONENTS & LOGIC]

1. DATA ARCHITECTURE (Modular Pipeline)
   - แยกส่วน Raw Data (.txt) ออกจาก Processing Logic (.py)
   - ใช้ JSON เป็นตัวกลาง (Intermediate Representation) เพื่อจัดเก็บ Metadata 
     เช่น เลขมาตรา, ประเภทกฎหมาย และเนื้อหาแยกส่วนกัน
   - ข้อดี: ทำให้ Dataset ขยายตัวได้ง่าย (Scalable) โดยไม่ต้องแก้ไข Code หลัก

2. THAI LEGAL TOKENIZER (Rule-based Approach)
   - ปัญหา: Tokenizer ทั่วไปมักตัดคำศัพท์กฎหมายเฉพาะทางผิดพลาด
   - แนวทางแก้ไข: ใช้ "Legal Compounds Protection" โดยการกำหนดคำสำคัญ 
     (เช่น 'สิทธิบัตรการประดิษฐ์', 'การละเมิดสิทธิ') และใช้ Placeholder 
     ป้องกันการถูกตัดแยกส่วนก่อนเริ่มกระบวนการ Tokenize ปกติ
   - ผลลัพธ์: รักษาความหมายเชิงบริบทของตัวบทกฎหมายได้แม่นยำกว่า

3. LEGAL ENTITY EXTRACTION (Regex-based)
   - พัฒนา Patterns เพื่อสกัด Entities สำคัญ:
     * STATUTE: เลขมาตรา และ พ.ร.บ.
     * PENALTY: บทลงโทษ (จำคุก/ปรับ) เพื่อใช้คำนวณ Severity ในอนาคต
     * ACTION: พฤติการณ์ที่เข้าข่ายการละเมิด

4. FEATURE REPRESENTATION
   - TF-IDF: ใช้ระบุ "Weight" ของคำในแต่ละมาตรา เพื่อหาจุดเด่นของแต่ละหมวดกฎหมาย
   - Co-occurrence Matrix: สร้างความสัมพันธ์เบื้องต้นระหว่างคำ (Contextual Linkage) 
     ก่อนจะอัปเกรดเป็น BERT Embeddings
